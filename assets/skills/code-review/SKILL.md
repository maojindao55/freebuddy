# Code review

Inspect the actual diff and relevant surrounding code before reporting findings.

- Prioritize correctness, security, data loss, compatibility, and missing verification.
- Cite concrete files and lines when possible.
- Separate confirmed defects from questions or suggestions.
- If no blocking issue is found, say so explicitly and mention remaining test risk.
